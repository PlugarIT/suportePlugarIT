Conversao de dados Folhamatic Postgres SQL

Para conversão de dados da Folhamatic SQL é obrigatório que o Postgres SQL esteja instalado na máquina de conversão nas versões 10 - 9.2

Precisamos do backup das pastas conforme descritas abaixo.

Para saber como faz o backup Folhamatic clique no link abaixo.

Clique aqui para SABER COMO FAZ O BACKUP FOLHAMATIC POSTGRE SQL.

Iniciando conversão Folhamatic Postgres SQL

  • Copie as duas pastas para a pasta E:\Bases\Folhamatic\SQL na máquina de conversão \\Contspcnv01 ou \\Contspcnv02.

  • Descompacte a pasta compactada (Backup_Folhaxxxxxxx.zip

  • Crie uma pasta com o código e apelido do cliente, esse será o nome do Database a ser criado

Execute o Postgres SQL para criar o DataBase do backup.

  • Dica: No arquivo de Backup SQL vem a versão do Postgres.

Para criarmos o DataBase selecione no Postgres a verão ao qual será criado o DataBase .

Clique no botão Direito no item Database e em seguida em New Database.

Digite o nome do Database e clique em ok.

Após criar o Database entre na pasta Conversao VC que esta na aréa de trabalho e execute o RestorePostgresFolhamatic.exe.

No Restore digitaremos os seguintes campos.

  • Pasta Bin PG: O local do Database.(Obs. Este local muda de acordo com a versão do Postgres, verificar qual o caminho correto no txt da pasta da Conversão VD.

  • Porta: O número da porta que utilizamos.

  • Data: O nome do Database.

  • Pasta Backup: O local que foi descompactado o Backup.

Clique em Run e aguarde restaurar o Database.

  • Antes de começar a conversão vá até a Pasta Phoenix e crie uma pasta Empresas e uma Geral.

  • Vá até a pasta Conversao VD que esta na aréa de trabalho e execute o CopiaDDF.bat para copiar as DDFs para a pasta Geral.

  • Este arquivo irá apagar todo o conteúdo da pasta Geral e copia as DDFs necessárias para a conversão.

Executando a cópia.

Após restaurar o Database entre na pasta Conversao VD que esta na aréa de trabalho e execute o MigradorVD.

Nessa tela iremos fazer algumas configurações.

Clicando no botão ao lado de SAGE_OFFICE_SQL Vai perguntar se deseja criar um novo layout Padrão

Clique em Sim.

Abrirá a próxima tela.

Configuramos os campos demarcados em vermelho.

  • Database é o Database criado.

  • Base de dados destino é o local onde será salvo a conversão.

  • Arquivos do cliente é onde se encontra o backup.

Agora clique em Importar.

Ao clicar em Importar O sistema dará a mensagem que importou as empresas com sucesso.

O sistema irá importar somente os apelidos das empresas, para importar os cadastros faremos o seguinte:

Entre na pasta da base do cliente, será criada uma Planilha com o nome de DeParaEmp.xlsx.

E uma Planilha com o nome de DeParaEmpNRel.xlsx.

Abra a planilha DeParaEmpNRel.xlsx

Agora abra a planilha DeParaEmp.xlsx.

E copie a primeira coluna da planilha anterior para a primeira e segunda coluna dessa planilha.

Coluna EMPNEW

Vai ficar conforme abaixo.

Na coluna EMPNEW podemos modificar o apelido da empresa para outro, ou se deixarmos em branco essa empresa não será convertida.

Agora volte para o Migrador e selecione os dados a serem convertidos.

Em seguida clique em Importar.

Aguarde a importação.
