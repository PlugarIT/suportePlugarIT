Status 86

A tabela de arquivos esta cheia, este status acontece no btrieve 6.15

ATENÇÃO

O Status 86 ao entrar no sistema, acontece porque excedeu a quantidade de Handles que foi definida na configuração do registro.

A primeira coisa a ser verificado é se a configuração esta correta no registro. Toda configuração do registro é importante, mas as que causam mais mesagens de erro são:

    Max Files=300

    Max Handles=600

    Max Transation=300

    Max Locks=300

Para solucionar entre na pasta em C:\CSS\Script e execute o arquivo Btrieve64bits.reg como administrador.

Para rodar o arquivo .reg, feche todos os sistemas e finalize no gerenciador de tarefas

Status 86 ao desinstalar o Btrieve 12 e voltar para o 6.15

Após desinstalar o Btrieve 12 e instalar o Btrieve 6.15 aparece a mensgem de Status 86 em alguma tabela do Robô

Isso acontece porque o registro não foi concluido com a configuração correta.

Para corrigir, primeiro certifique se a configuração esta correta, para isso digite no menu Iniciar do Windows REGEDIT e dê enter.

Expanda o item HKEY_LOCAL_MACHINE e em seguida SOFTWARE conforme abaixo.

Em seguida abra o item WOW6432Node e Btrieve Technologies

Expanda o item Btrieve Technologies até o último item SETTINGS

Verifique do lado direito os itens abaixo estão conforme os valores da imagem.

    Max Clients=30

    Max Communication Buffer Size=16384

    Max Files=300

    Max Handles=600

    Max Locks=300

    Max Transactions=18

Caso esteja com valores menores do que aparece na Imagem, apague o Item Btrieve Tecnologies para poder criar outro com os valores correto.

Para criar essa configuração no Registro do Windows podemos reinstalar um dos sistemas ou rodar o arquivo .Reg para exportar essa chave para o Registro.

O arquivo para exportar encontra-se em: C:\CSS\Script Btrieve64bits.reg

    DICA 1: Verificar se tem 2 Geradores aberto e verifique também como esta o registro do btrieve (REGEDIT), caso não esteja configurado corretamente entre na pasta C:\CSS\Script e rode o arquivo Btrieve64bits.reg como administrador.

    DICA 2: Caso os itens acima estejam corretos tentar renomear o arquivo CNCUSUAR.BTR na pasta Geral.
