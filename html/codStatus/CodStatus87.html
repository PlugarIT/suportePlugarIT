Status 87

A tabela de handles está cheia

The handle table is full

    Este código de status é obsoleto nas versões 2000i e superiores do Pervasive.SQL.

    A tabela de handles é administrada dinamicamente pelo sistema.

    Você tentou abrir mais handles que o permitido pela configuração do MicroKernel ou o MicroKernel tentou abrir mais arquivos que o permitido pelo sistema operacional.

    • Consulte a documentação do sistema operacional para fazer com que ele permita mais handles.

      • Saber os seguintes detalhes sobre os requisitos do MicroKernel quanto aos handles do sistema operacional pode ser útil: quando o mesmo arquivo é aberto várias vezes, o MicroKernel usa apenas um handle do sistema operacional. Entretanto, se o arquivo estiver no formato v6.x ou superior e estiver sendo compartilhado por meio do modo MEFS, o MicroKernel usa um handle adicional para o arquivo .LCK associado.

    • Se o arquivo estiver no formato v5.x, o MicroKernel pode usar um handle adicional para o arquivo .PRE. Além disso, se o arquivo (em qualquer formato) estiver em modo de operação contínua, o MicroKernel usa um handle adicional para o arquivo diferencial. Se o arquivo tem extensões, o MicroKernel usa um handle do sistema operacional para cada arquivo de extensão.

    • Em ambientes Btrieve v6.15 em DOS ou em Windows NT 4.0, este código de status pode ocorrer quando mais de 15 arquivos são abertos por um aplicativo DOS sendo executado em Windows NT. Há duas soluções possíveis:

    • A configuração de handles de arquivo do Btrieve, em BTI.CFG, pode estar incorreta. Procure as opções de handles de arquivo em BTI.CFG (opções /h: e /f:) e aumente seus valores.

    • Verifique a configuração "file=" em CONFIG.NT, que está no diretório WINNT\SYSTEM32, e aumente o valor.

    Nota: O valor padrão é 20.

    Nota 2: Esse valor foi colocado a vários anos atrás, hoje o minímo esta em 600
