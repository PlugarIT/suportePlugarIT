Status 12

O MicroKernel não pode encontrar o arquivo especificado

The MicroKernel cannot find the specified file

Verifique se o arquivo existe e se o arquivo correto foi especificado.

  • Verifique o parâmetro de buffer de chave para certificar-se de que o nome termine com um espaço ou com um zero binário.
  • Ao acessar um arquivo em um servidor, certifique-se de possuir direitos de FILE SCAN para o diretório onde o arquivo está. Eventualmente, uma chave danificada pode causar este código de status. Remova a chave e adicione-a novamente.
  • Em ambientes Novell NetWare 4.x, procure arquivos que estejam com o atributo hidden (oculto) no nível do sistema operacional e remova esse atributo.
  • Isto não ocorre com NetWare 5.
  • Uma conexão ODBC pode receber este código em NetWare 4.x com o sistema de grupo de trabalho do Pervasive.SQL. Este código de status não ocorria antes do Service Pack 2a do Pervasive.SQL 2000. Ele ocorre porque o NetWare 4.x reporta nomes de diretório longos em um formato diferente do NetWare versões 3.x e 5.x e do ambiente Win32.
  • A solução da Novell para este problema é um arquivo chamado Mixmod6.exe. Procure por esse arquivo no site da Novell (http://www.novell.com/).
  • Este código de erro também pode ser retornado se o arquivo DBNames.CFG for removido (pelo utilitário de limpeza do Pervasive.SQL, por exemplo) e os nomes de fontes de dados (Data Source Names, DSNs) antigos que faziam referência a esse arquivo não forem removidos da configuração do ODBC.
  • Você tentou abrir um arquivo local usando um sistema de grupo de trabalho que não é o sistema de Gateway destinado ao arquivo. Vá para o diretório onde está o arquivo que você tentou abrir. Verifique se o arquivo ~pvsw~.loc desse diretório está marcado como somente leitura. Se estiver, remova o atributo de somente leitura.
  • Pode ser que o código de status 12 seja retornado e o nome do arquivo nos DDFs seja X$FILE.DDF.
  • Quando se usa um utilitário do Btrieve para examinar o arquivo file.ddf, os nomes de localização para os arquivos de dicionário são x$file.ddf, x$field.ddf e x$index.ddf, em lugar dos padrões, que são file.ddf, field.ddf e index.ddf. Os DDFs foram criados por um utilitário de criação de DDF não suportado, que usou DDL Services (DDLSVCS.DLL). DDL Services tem um defeito conhecido que faz com que a tabela de sistema seja preenchida com dados incorretos. Para corrigir este erro, recrie os DDFs usando uma ferramenta mais nova (SQL Data Manager/Pervasive Control Center).
  • Pode ocorrer o status 12 quando um nome de arquivo ou um caminho contendo espaços for aberto em Windows 9x, Windows NT ou Windows 2000. O suporte a nomes longos do NetWare está disponível somente no MicroKernel, não em utilitários NLM como BUTIL.NLM. Arquivos de dados do Btrieve podem ser acessados com nomes longos, mas outros arquivos não. Por exemplo, no seguinte comando NLM:
    • BUTIL -CREATE O arquivo de dados do Btrieve pode ter um nome longo porque é manipulado pelo MKDE. Entretanto, o nome do arquivo precisa ser curto (BUTIL.NLM, por exemplo) porque BUTIL não aceita nomes longos. A solução é habilitar a opção de configuração do cliente Application characteristics Embedded Spaces.
