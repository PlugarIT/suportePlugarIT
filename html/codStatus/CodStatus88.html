Status 88

O aplicativo encontrou um erro de modo incompatível

The application encountered an incompatible mode error

O MicroKernel retorna este código de status nas seguintes situações:

  • Se um aplicativo tiver aberto um arquivo em modo exclusivo, todos os demais aplicativos recebem este código de status quando tentam abrir o mesmo arquivo em qualquer modo.

  • Se um aplicativo tiver aberto um arquivo em qualquer modo diferente de exclusivo, todos os demais aplicativos recebem este código de status quando tentam abrir o mesmo arquivo em modo exclusivo.

  • Se você estiver usando o modo de operação contínua do MicroKernel:

  • Você tentou retirar um arquivo do modo de operação contínua, mas o arquivo não estava no modo de operação contínua.

  • Você tentou retirar um arquivo do modo de operação contínua, mas outro cliente colocou o arquivo em modo de operação contínua.

  • Você tentou colocar dois arquivos que têm o mesmo nome e extensões diferentes em modo de operação contínua.

  • Você tentou colocar um arquivo no modo de operação contínua, mas o arquivo já estava no modo de operação contínua.

  • Os arquivos estavam no modo de operação contínua e o servidor caiu. Agora, quando se tenta retirar os arquivos da operação contínua, ocorre o código de status 88.

No último caso descrito acima, uma vez que o servidor tenha caído, o sistema Btrieve não sabe quais arquivos estavam em operação contínua. O código de status 88 é retornado como conseqüência dessa condição.

Para retirar os arquivos da operação contínua, é necessário abri-los antes de encerrar o modo de operação contínua. Quando o arquivo é reaberto, o sistema do Btrieve verifica que o flag de operação contínua está ligado e procura o arquivo diferencial. Nesse ponto, o arquivo diferencial é efetivado (roll-in).

Para efetivar um arquivo diferencial existente, o arquivo de dados associado precisa ser aberto. Você pode abrir o arquivo com um utilitário como o function executor ou com o aplicativo que usa o arquivo. Se esse aplicativo abre e fecha o arquivo repetidas vezes, recomenda-se abrir o arquivo com o function executor porque efetivar o arquivo diferencial é uma tarefa de prioridade muito baixa. A efetivação foi desenvolvida desse modo para que o arquivo possa ser usado enquanto ocorre o processo. Se o aplicativo fechar o arquivo sem que a efetivação tenha terminado, a efetivação será reiniciada quando o arquivo for reaberto. Sendo uma tarefa de baixa prioridade, o processo de efetivação pode demorar um pouco. O sistema apaga o arquivo diferencial quando a efetivação é concluída.
