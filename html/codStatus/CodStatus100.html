Status 100

A sessão não é mais válida

No Cache Buffers Available

    Este código indica que o Microkernel usou os buffers de cache alocados durante sua inicialização. Este código de status pode ocorrer se o aplicativo faz um grande número de operações de escrita (inserções, atualizações e exclusões) em uma transação de usuário. A implementação atual do MKDE exige que trodas as páginas modificadas fiquemno cache até que a transação seja completada.

    Nota: em computador com pouca meória disponível, pode não ser possível completar com sucesso transações muito grandes, com milhares de operações de escrita.

    Se você é desenvolvedor, pode modificar o aplicativo para que faça o Commit das transações com mais frequência, para que menos páginas modificadas permaneçam no cache. A solução mais comum para este problema é aumentar a opção de configuração Cache Allocation e recarregar o Microkernel

    A partir do Pervasive.SQL 8, a alocação de cache pode sr administrada dinamicamente pelo Microkernel. Se este for o caso e mesmo assim você receber o código 100, é necessário instalar mai memória.

    Para aumentar a opção de Cache Allocation:

    • Clique em iniciar\Programas\Pervasive.SQL V8\Control Center.
    • ExpandaPervasive.SQL V8 Enginese expanda o banco de dados desejado.
    • Expanda Configuration, depois clique em Performance Tunning.
    • No quadro do lado direito, clique duas vezes em Cache Allocation Size.
    • Aumente o valo Current digitando o número de bytes de memória para alocar para o cache. Nota: o padrão é dinâmico.
    • Encerre o utilitário de configuração e reinicie os sistemas para que as novas configurações sejam efetivadas.
