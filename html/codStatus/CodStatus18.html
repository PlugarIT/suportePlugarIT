Status 18

O disco está cheio

The disk is full

O MicroKernel pode retornar este código de status nas seguintes situações:

  • O disco está cheio e o MicroKernel não pode expandir o arquivo para acomodar novos registros. Apague os arquivos desnecessários.
  • Não há espaço suficiente para acrescentar uma nova página ao arquivo de dados.
  • Não há espaço suficiente para o arquivo de pré-imagem. Se os seus arquivos estão em um formato anterior à v6.0, o tamanho do arquivo de pré-imagem aumenta enquanto dura uma transação. Se você receber este código de status, reduza o número de operações da transação ou libere mais espaço em disco.
  • Para arquivos localizados em um servidor NetWare, o nome do proprietário do arquivo no NetWare não é mais válido e o aplicativo tentou inserir ou atualizar registros no arquivo, provocando sua expansão. Neste caso, o MicroKernel retorna este código de status quando precisa acrescentar uma página ao arquivo independentemente do espaço disponível no disco. Para conferir o nome do proprietário, use o utilitário NDIR do NetWare. Para acrescentar um nome de proprietário, use FILTER (um utilitário em modo texto do NetWare) ou o utilitário gráfico NetWare Administrator.
  • Em alguns ambientes, é possível limitar o espaço em disco disponível para cada usuário. Este código de status indica que o aplicativo tentou expandir um arquivo de dados além do limite de espaço estabelecido para o proprietário do arquivo.
  • Você tentou ler ou modificar um arquivo que não tinha sido fechado adequadamente após um erro de disco cheio. Certifique-se de que todos os aplicativos que estavam usando o arquivo no momento em que ocorreu o erro de disco cheio fecharam o arquivo com sucesso.
  • Se um cliente conectado a um servidor do Pervasive.SQL encontrar este código de status, outros clientes que estiverem executando operações somente de leitura no mesmo disco também podem receber um status diferente de zero.
