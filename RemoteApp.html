Instalação

Instalar os serviços de desktop remoto (RDS) no Windows Server 2019 parece complicado, mas na verdade é bem fácil. Nesse artigo, veremos como instalar esses serviços em um ambiente de domínio que requer dois servidores.

Pré-requisitos

Há apenas dois passos necessários antes de podermos instalar o RDS. Eles são:

  • Todos os servidores estão no domínio
  • Há pelo menos dois servidores disponíveis

O motivo de precisarmos de dois servidores é que a função de Licenciamento do RD irá para o segundo servidor, em vez de manter tudo no mesmo servidor. Isso é considerado uma das melhores práticas da Microsoft. Nesse tutorial, usamos o controlador de domínio como o servidor de licença do RD, o que não seria considerado uma das melhores práticas. O motivo disso nesse tutorial é manter a instalação simples.

Instalação das funções básicas dos serviços de desktop remoto

Nessa primeira série de instalação, vamos adicionar as seguintes funções ao nosso servidor RDS primário:

  • Broker de conexão RD
  • Acesso à web RD
  • Host de sessão RD
Etapas de instalação

1 - No gerenciador de servidores do servidor RDS primário que estamos instalando, abra o Assistente para Adicionar Funções e Recursos e selecione a instalação dos Serviços de Desktop Remoto.

2 - Nesse tutorial, vamos escolher o Início Rápido, mas se você precisa de mais controle sobre o processo de instalação, pode usar a Implantação Padrão para modificar mais opções durante a instalação.

3 - A seguir, escolheremos Implantação em desktop baseada em sessão, pois esse é um modelo de sessão para RemoteApp e desktop normalmente usado em implantações de RDS mais tradicionais.

4 - Na Seleção de servidor, escolha o servidor em que estamos instalando o RDS.

5 - Escolha Reiniciar o servidor de destino automaticamente se necessário e clique em Ativar para iniciar a instalação.

6 - Verifique se todas as funções foram instaladas antes de avançar para as próximas etapas.

Adicionar servidor secundário

Para esse tutorial, vamos usar o Servidor de Domínio como nosso servidor de Licenciamento de RD. Porém, para instalar facilmente essa função, podemos adicionar outro servidor ao Gerenciador de Servidores.

1 - Para adicionar o servidor secundário, clique com o botão direito em Todos os Servidores, escolha Adicionar Servidores e então selecione o servidor no Active Directory.

2 - Navegue até os Serviços de Desktop Remoto e clique no sinal de adição verde para o Licenciamento de RD.

3 - A tela Adicionar Servidores de Licenciamento de RD será exibida e permitirá que você adicione o servidor secundário como destino da função de Licenciamento de RD.

4 - Clique em Adicionar para instalar a função no servidor secundário.

5 - Verifique se a instalação foi concluída, conferindo se o símbolo de adição verde foi substituído pelo item apropriado em Licenciamento de RD.

Adicionar função de Gateway de RD

Por fim, precisamos adicionar a Função de Gateway de RD para o nosso servidor RDS primário.

1 - Na tela Serviços de Desktop Remoto, clique no sinal de adição verde sobre Gateway de RD.

2 - Selecione o servidor RDS primário a ser usado para a instalação dessa função.

3 - Indique o certificado SSL autoassinado com um nome de domínio totalmente qualificado.

4 - Clique em Avançar e depois em Adicionar para instalar a função em nosso servidor RDS primário.

Configurar propriedades de implantação

Agora que todas as funções foram instaladas, podemos configurar as propriedades de implantação.

1 - Vá até a tela Serviços de Desktop Remoto e, no menu suspenso Tarefas, clique em Editar Propriedades de Implantação.

2 - Deixe as configurações padrão na tela Gateway de RD e clique no item de menu Licenciamento de RD.

3 - Escolha Por usuário na tela Licenciamento de RD. Você pode escolher qualquer opção, mas para os fins desse tutorial, selecionaremos Por usuário.

4 - Observe a URL da tela Acesso à Web RD, pois ela será usada mais tarde para acessar as aplicações implantadas.

5 - Para fins de teste, você pode deixar os certificados Não Configurados, e por fim clique em OK para salvar a configuração de implantação.

Se quiser configurar um certificado, você terá que fazer isso para cada um dos serviços de função individualmente

Verificar Serviços de Desktop Remoto

Por padrão, uma QuickSessionCollection foi criada após a instalação, contendo a Calculadora, o WordPad e o Paint como RemoteApps. Isso pode ser usado para testar a implantação RDP.

1 - Navegue até a URL do IIS originalmente localizada em Acesso à web RD, ou você pode usar https://localhost/rdweb/se estiver localizado no próprio servidor RDP, para testar o IIS.

2 - Faça login na sessão de RDS do IIS usando credenciais do domínio.

3 - Por fim, inicie uma conexão remota. Pode ser uma definida por você ou um RemoteApp padrão.

São necessárias várias etapas para implantar os Serviços de Desktop Remoto, mas depois é fácil configurá-lo e usá-lo. Os RemoteApps oferecem muita flexibilidade, assim como a capacidade de definir coleções de conexões RDP que podem ser oferecidas aos usuários.
