Status 29

O tamanho de chave é inválido

The key length is invalid

    O MicroKernel retorna este código de status nas seguintes situações:

    O tamanho da chave especificada é menor que 1 ou maior que 255 bytes. O tamanho de um segmento de chave precisa estar de acordo com o tipo da chave se o tipo da chave tem um tamanho associado a ele (por exemplo, uma chave do tipo inteiro precisa ter um comprimento divisível por dois). Cada uma das páginas de chave do arquivo precisa ter um tamanho suficiente para conter pelo menos oito chaves.

    Se o tamanho da página não for suficiente para conter oito ocorrências do tamanho especificado para a chave (mais extras), será necessário aumentar o tamanho de página do arquivo ou diminuir o tamanho da chave.

    É necessário um byte a mais de espaço de armazenamento para o indicador de nulo da coluna. Este erro ocorre durante a execução de um SQL CREATE INDEX ou durante a criação de um SQL PRIMARY KEY ou FOREIGN KEY se o índice ou a chave se referirem a uma coluna CHAR nula de 255 caracteres (ou a uma coluna VARCHAR de 254 caracteres). O byte adicional faz com que o tamanho real do índice seja um byte maior, ou seja, 256 bytes. Para corrigir o erro, reduza o tamanho da coluna ou crie a coluna como NOT NULL e tente novamente. Para uma chave estrangeira (foreign key), se o tamanho da coluna for reduzido será necessário reduzir a coluna que faz a referência, bem como a coluna referida.

    Você tentou criar um arquivo do Btrieve com mais de um índice. Isto pode ocorrer no Btrieve v6.15 em ambiente Windows95/NT. O problema pode ser reproduzido desenvolvendo-se um aplicativo de 32 bits usando o Microsoft Visual Basic e é causado por variáveis long alinhadas em endereços de word pares.

    A solução é evitar o uso de variáveis long no tipo KeySpec. Use uma string de 4 bytes em lugar de uma variável long na definição do tipo. Depois, converta a string para long.
