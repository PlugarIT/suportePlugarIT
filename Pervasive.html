O Pervasive e um banco de dados Relacional

  • Pervasive PSQL é um banco de dados relacional desenvolvido pela Pervasive Software, usado principalmente como banco de dados embutido. Originalmente, o nome do produto era Btrieve e foi escrito por Doug e Nancy Woodward.

  • A Btrieve foi adquirida pela Novell em 1987, mas desmembrou-se em 1994 como uma nova empresa chamada Btrieve Technogies (liderada por Doug e Nancy Woodward e Ron Harris). A Btrieve Technologies foi renomeada para Pervasive Software. A maior força da Pervasive é a confiabilidade. Está continuamente no mercado há 30 anos e os clientes não relatam problemas operacionais com ele. Enquanto outros bancos de dados são mais conhecidos, o Pervasive, por ser um banco de dados embutido, recebe menos atenção do público. No entanto, é extremamente popular e bem-sucedido. A Pervasive Software tem 3500 parceiros em todo o mundo, incluindo 200 OEMs.

  • O Pervasive PSQL tem dois mecanismos de banco de dados - mecanismo relacional (permite que você use SQL para trabalhar com o banco de dados) e mecanismo transacional (também conhecido como MicroKernel Database Engine ou MKDE, permite que você vá diretamente para o banco de dados e ignore completamente o SQL). O MKDE garante um estado válido do banco de dados mesmo em caso de perda de energia.
